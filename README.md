# 提交工作台

一套完整的提交单管理模块，支持新建提交单、暂存、正式提交、撤回、再发起的全流程管理。数据本地持久化，支持 CSV 双向流转，覆盖跨重启恢复、旧草稿处理、重复提交拦截等真实场景。

## 功能特性

### 核心流程

- **新建提交单**：选择设备、模板、日期，快速创建草稿
- **暂存草稿**：本地保存，随时继续编辑，自动会话恢复
- **正式提交**：四层校验（必填项、模板版本、状态门禁、重复检测），生成提交凭据
- **撤回修改**：提交后可撤回，保留审计日志
- **重新发起**：撤回后修改再提交，完整追溯流程

### 模块能力

- **任务列表**：按待办、进行中、已撤回、已完成四分区管理
- **详情侧栏**：完整信息展示，快捷操作入口
- **流转时间线**：完整状态变更历史，清晰追溯
- **操作日志**：所有操作全记录，便于审计追溯
- **凭证视图**：提交凭据留存，快照哈希防篡改

### 数据能力

- **CSV 双向流转**：导出后可再导回系统，完整数据回灌
- **记录 ID 保留**：导入保留原始记录 ID，便于追溯
- **冲突检测**：ID 重复、同日同设备、内容完全相同三种冲突检测
- **字段兼容**：自动处理字段缺失和版本差异，支持自动迁移
- **真实场景**：跨重启恢复、旧草稿处理、重复提交拦截

## 技术栈

- 前端：React 18 + TypeScript + Vite
- 状态管理：Zustand
- 本地存储：IndexedDB (idb)
- UI 框架：Tailwind CSS
- 图标：Lucide React
- 后端：Express + Node.js
- 测试：Vitest + Testing Library

## 快速上手：5 分钟体验完整流程

### 第 1 步：进入工作台

打开应用后，首页会展示提交工作台概览。点击「进入工作台」按钮，或通过底部导航进入提交工作台页面。

### 第 2 步：新建提交单

1. 点击页面顶部的「新建提交单」按钮
2. 在弹窗中选择：
   - 设备：选择要提交的设备
   - 模板：选择适用的提交模板
   - 日期：默认为今日，可调整
3. 点击「创建草稿」，系统会创建一条草稿记录
4. 草稿会出现在「待办列表」分区

### 第 3 步：暂存草稿

1. 在待办列表中点击刚创建的记录，打开详情侧栏
2. 点击「编辑」按钮，进入提交单编辑页面
3. 填写模板中的各项内容
4. 内容会自动保存为草稿（3 秒防抖自动保存）
5. 也可以手动点击「保存草稿」按钮

### 第 4 步：正式提交

1. 填写完成后，回到工作台详情侧栏
2. 点击「提交」按钮
3. 系统会自动进行四层校验：
   - 必填项校验：检查所有必填字段是否已填写
   - 模板版本校验：确认使用的是最新模板版本
   - 状态门禁校验：当前状态是否允许提交操作
   - 重复提交校验：同设备同日是否已有提交中的记录
4. 校验通过后，记录状态变为「已提交待同步」
5. 生成提交凭据：包含凭据编号、操作时间、来源设备、数据哈希、快照 ID

### 第 5 步：撤回修改

1. 在「进行中」分区找到已提交的记录
2. 点击打开详情侧栏
3. 点击「撤回」按钮
4. 确认撤回操作
5. 记录状态变为「已撤回待重提」
6. 撤回操作会被记录到审计日志中

### 第 6 步：重新发起

1. 在「已撤回」分区找到撤回的记录
2. 点击打开详情侧栏
3. 可先点击「编辑」修改内容
4. 点击「重新发起」按钮
5. 记录再次进入提交状态，生成新的提交凭据
6. 完整的提交-撤回-再提交流程可在时间线中查看

## 工作台分区说明

提交工作台将记录分为四个分区，便于分类管理：

| 分区 | 包含状态 | 说明 |
|------|----------|------|
| 待办列表 | 草稿、续办中 | 需要处理的提交任务，可编辑和提交 |
| 进行中 | 待同步、冲突 | 已提交等待同步，或有冲突需处理 |
| 已撤回 | 已撤回 | 已从同步队列撤回，可重新发起 |
| 已完成 | 已同步 | 已成功同步到服务器的记录 |

## 详情侧栏功能

点击记录后，右侧会展开详情侧栏，包含以下模块：

### 基础信息

- 记录 ID、设备信息、模板信息
- 巡检日期、巡检员、照片数
- 提交次数、撤回次数

### 提交凭据

- 凭据编号（唯一标识）
- 操作时间
- 来源设备信息
- 操作人
- 快照 ID
- 数据哈希（用于校验数据完整性）
- 可导出提交快照

### 审计日志

- 撤回等关键操作的审计记录
- 包含操作时间、操作人、操作结果

### 状态变更轨迹

- 完整的状态变更时间线
- 每次变更的时间、操作人、备注
- 清晰展示提交单的完整生命周期

### 操作日志

- 针对该记录的所有操作记录
- 包含提交、撤回、迁移等操作

### 时间戳

- 创建时间、最后更新
- 首次提交、最近提交
- 最近撤回、同步时间
- 冲突解决时间

## CSV 导入导出

### 导出 CSV

1. 进入提交工作台
2. 使用筛选器选择要导出的记录（时间段、设备、状态）
3. 点击「导出 CSV」按钮
4. 下载 CSV 文件

导出内容包含：
- 记录 ID、设备信息、模板信息
- 状态、异常等级、提交次数、撤回次数
- 提交内容（JSON 格式）、照片列表
- 所有时间戳（创建、更新、提交、撤回、同步）
- 冲突信息、凭据编号、来源设备
- 状态历史数、快照数、审计日志数

### 导入 CSV（回灌）

1. 进入提交工作台
2. 点击「导入 CSV」按钮
3. 选择之前导出的 CSV 文件
4. 系统自动处理导入

导入能力：
- **记录 ID 保留**：导入的记录保留原始 ID，便于追溯
- **冲突检测**：
  - ID 重复：已存在的记录会被跳过
  - 同日同设备：检测同设备同日是否已有提交记录
  - 内容相同：检测数据哈希是否完全一致
- **字段兼容**：
  - 自动解析 JSON 格式的 values 和 photos 字段
  - 缺失字段自动填充默认值
  - 模板版本不匹配时可自动迁移
- **结果报告**：成功数、跳过数、失败数、警告信息

## 真实场景覆盖

### 跨重启恢复

- 刷新或关闭浏览器后重新打开，自动恢复草稿
- 会话状态保存在 IndexedDB 中
- 首页和工作台都会显示恢复提示
- 支持恢复上次编辑的现场

### 旧草稿处理

- 模板版本更新后，自动检测基于旧版模板的草稿
- 工作台顶部显示旧草稿提示横幅
- 支持一键迁移至最新模板版本
- 迁移时新字段自动填充默认值
- 迁移后状态重置为草稿，可继续编辑

### 重复提交拦截

- 提交前自动检测同设备同日是否已有提交中的记录
- 防止重复提交造成数据冗余
- 按钮会显示禁用原因提示
- 可在详情中查看具体原因

## 状态说明

| 状态 | 说明 | 可执行操作 |
|------|------|------------|
| 本地草稿 | 仅保存在本设备，可编辑 | 编辑、提交 |
| 已恢复续办 | 上次会话现场已恢复 | 编辑、提交 |
| 已提交待同步 | 已进入同步队列，等待上传 | 撤回 |
| 冲突待处理 | 多端数据冲突，需人工处理 | 处理冲突 |
| 已撤回待重提 | 已从同步队列撤回，保留审计日志 | 编辑、重新发起 |
| 已同步记录 | 已上传服务器，可跨设备查看 | 查看详情 |

## 按钮门禁说明

所有操作按钮都受状态门禁控制，按钮禁用时会显示具体原因：

| 禁用原因 | 说明 |
|----------|------|
| 模板版本已更新 | 草稿基于旧版模板，请先迁移 |
| 存在必填项未填写 | 有必填字段未填写，无法提交 |
| 存在未解决的冲突 | 有数据冲突需先处理 |
| 同设备同日已有提交 | 该设备今日已有提交中的记录 |
| 仅"已提交"状态可撤回 | 当前状态不允许撤回操作 |
| 仅"已撤回"状态可重新发起 | 当前状态不允许重新发起 |
| 仅草稿或已撤回状态可编辑 | 当前状态不可编辑 |

## 常见问题

**Q: 数据会丢失吗？**

A: 所有数据保存在浏览器的 IndexedDB 中，刷新页面或关闭浏览器后重新打开，数据不会丢失。但清除浏览器数据会导致本地数据丢失，建议定期导出备份。

**Q: 如何导出 CSV 再导回？**

A: 在提交工作台中，先使用筛选器选择要导出的记录，点击「导出 CSV」下载文件。需要导回时，点击「导入 CSV」选择刚才下载的文件即可。系统会自动处理冲突和字段兼容。

**Q: 模板更新后旧草稿怎么办？**

A: 工作台会自动检测基于旧版模板的草稿，并在顶部显示提示。点击「迁移至最新模板」按钮，系统会自动填充新字段的默认值，迁移完成后可继续编辑和提交。

**Q: 同设备同日能提交多条记录吗？**

A: 默认会拦截重复提交，防止误操作。如果确实需要提交多条（例如上午和下午各一次），可以在编辑页面调整日期或使用不同的记录 ID。

**Q: 撤回后数据会丢失吗？**

A: 不会。撤回操作只是将记录从同步队列移回草稿区，所有数据都会保留，并且会生成审计日志记录撤回操作。撤回后可以修改内容后重新发起提交。

**Q: 提交凭据有什么用？**

A: 提交凭据是每次提交的唯一凭证，包含凭据编号、数据哈希、快照信息等。可用于事后追溯、数据校验、审计等场景。
